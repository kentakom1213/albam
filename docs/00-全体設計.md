以下の方針で，バックエンドを段階的に実装していくとよいです．前提として，このアプリは「Go 製 API + Astro 製テーマ」に分離するセルフホスト型アルバム管理ツールとして設計します．

## 0. 最終的に作りたいもの

```txt
指定したディレクトリ以下の画像を自動で読み取り，
ディレクトリ構造をアルバム階層として扱い，
API 経由で画像・アルバム・メタデータを返す Go バックエンド
```

最終構成は次のようにします．

```txt
albam/
├── cmd/
│   └── albam/
│       └── main.go
├── internal/
│   ├── config/
│   ├── model/
│   ├── scanner/
│   ├── indexer/
│   ├── storage/
│   ├── imageproc/
│   ├── api/
│   └── watcher/
├── migrations/
├── data/
│   ├── albam.db
│   └── cache/
├── albums/
├── albam.example.yml
└── go.mod
```

## 1. Go プロジェクトを作る

最初に，CLI として起動できる最小構成を作ります．

```bash
mkdir albam
cd albam
go mod init github.com/kentakom1213/albam
mkdir -p cmd/albam internal/scanner internal/model
touch cmd/albam/main.go
```

最初の目標は，次のコマンドが動くことです．

```bash
go run ./cmd/albam index ./albums
```

## 2. ディレクトリ以下の画像を列挙する

まずは DB も API も作らず，画像ファイルを再帰的に列挙します．

担当パッケージは `internal/scanner` です．

```txt
internal/scanner/
└── scanner.go
```

ここで扱う型は次のようにします．

```go
type AssetFile struct {
	Path     string
	RelPath  string
	Filename string
	Ext      string
	Size     int64
	ModTime  time.Time
}
```

実装する関数はこれです．

```go
func Scan(root string) ([]AssetFile, error)
func IsSupportedImage(path string) bool
```

対応する拡張子は，最初はこれくらいで十分です．

```txt
.jpg
.jpeg
.png
.webp
```

この段階の完成条件は，次のように画像一覧が出ることです．

```txt
trip-kyoto/img001.jpg
trip-kyoto/img002.jpg
family/2026/photo.jpg
```

## 3. ディレクトリ構造からアルバム構造を作る

次に，画像ファイルの親ディレクトリをアルバムとして扱います．

たとえば，次のように対応させます．

```txt
albums/trip-kyoto/img001.jpg
  -> album: trip-kyoto

albums/family/2026/photo.jpg
  -> album: family/2026
```

担当パッケージは `internal/indexer` です．

```txt
internal/indexer/
└── indexer.go
```

モデルは `internal/model` に置きます．

```go
type Album struct {
	ID       int64
	ParentID *int64
	Path     string
	Slug     string
	Title    string
}

type Asset struct {
	ID       int64
	AlbumID  int64
	Path     string
	Filename string
	Ext      string
	Size     int64
	ModTime  time.Time
}
```

この段階では，まだ `ID` は仮で構いません．DB に保存する前なので，`Path` をキーにして扱えば十分です．

実装する関数は次のようなものです．

```go
func BuildLibrary(files []scanner.AssetFile) (*Library, error)
```

`Library` は，アルバム一覧と画像一覧をまとめた型です．

```go
type Library struct {
	Albums []model.Album
	Assets []model.Asset
}
```

## 4. SQLite に保存する

ここから永続化します．画像本体は DB に保存せず，ファイルパスとメタデータだけを保存します．

担当パッケージは `internal/storage` です．

```txt
internal/storage/
├── db.go
├── album.go
└── asset.go
```

最小スキーマは次のようにします．

```sql
CREATE TABLE albums (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    parent_id INTEGER,
    path TEXT NOT NULL UNIQUE,
    slug TEXT NOT NULL,
    title TEXT NOT NULL,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
);

CREATE TABLE assets (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    album_id INTEGER NOT NULL,
    path TEXT NOT NULL UNIQUE,
    filename TEXT NOT NULL,
    ext TEXT NOT NULL,
    size_bytes INTEGER NOT NULL,
    file_mtime DATETIME NOT NULL,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    FOREIGN KEY (album_id) REFERENCES albums(id)
);
```

この段階の目標は，次のコマンドです．

```bash
go run ./cmd/albam index ./albums
```

これにより，画像ディレクトリをスキャンして，`SQLite` に `albums` と `assets` が保存される状態にします．

## 5. HTTP API を作る

次に，DB の内容を HTTP API で返します．

担当パッケージは `internal/api` です．

```txt
internal/api/
├── server.go
├── album_handler.go
└── asset_handler.go
```

最初に作る API はこれだけでよいです．

```txt
GET /api/albums
GET /api/albums/{id}
GET /api/albums/{id}/assets
GET /api/assets/{id}
```

レスポンス例です．

```json
{
  "id": 1,
  "path": "trip-kyoto",
  "slug": "trip-kyoto",
  "title": "trip-kyoto"
}
```

画像一覧は次のように返します．

```json
[
  {
    "id": 10,
    "album_id": 1,
    "path": "trip-kyoto/img001.jpg",
    "filename": "img001.jpg",
    "url": "/api/assets/10/file"
  }
]
```

この段階の目標は，次のコマンドです．

```bash
go run ./cmd/albam serve
```

そして，ブラウザまたは `curl` で次が見える状態にします．

```bash
curl http://localhost:8080/api/albums
```

## 6. 画像ファイルを配信する

次に，画像ファイル本体を返す API を作ります．

追加する API はこれです．

```txt
GET /api/assets/{id}/file
```

実装では，DB から画像の相対パスを取得し，設定された `library.root` と結合して `http.ServeFile` で返します．

注意点として，パストラバーサル対策を必ず入れます．

```go
func ResolveAssetPath(root, relPath string) (string, error)
```

この関数では，最終的なパスが必ず `root` 以下に収まっていることを確認します．

## 7. サムネイル生成を追加する

元画像を一覧画面でそのまま使うと重いので，サムネイルを生成します．

担当パッケージは `internal/imageproc` です．

```txt
internal/imageproc/
└── thumbnail.go
```

キャッシュディレクトリは次のようにします．

```txt
data/cache/
└── thumbnails/
    ├── asset-1.webp
    ├── asset-2.webp
    └── asset-3.webp
```

DB には `asset_variants` を追加します．

```sql
CREATE TABLE asset_variants (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    asset_id INTEGER NOT NULL,
    kind TEXT NOT NULL,
    path TEXT NOT NULL,
    width INTEGER NOT NULL,
    height INTEGER NOT NULL,
    size_bytes INTEGER NOT NULL,
    created_at DATETIME NOT NULL,
    UNIQUE(asset_id, kind),
    FOREIGN KEY (asset_id) REFERENCES assets(id)
);
```

追加する API はこれです．

```txt
GET /api/assets/{id}/variants/thumb
GET /api/assets/{id}/variants/preview
```

最初は `thumb` だけでよいです．

## 8. 設定ファイルを導入する

ここまでできたら，コマンドライン引数ではなく `albam.yml` で設定できるようにします．

担当パッケージは `internal/config` です．

```txt
internal/config/
└── config.go
```

設定ファイル例です．

```yaml
server:
  host: 0.0.0.0
  port: 8080

library:
  root: ./albums

database:
  path: ./data/albam.db

cache:
  dir: ./data/cache

images:
  thumbnail_width: 512
  preview_width: 1600
```

起動コマンドは次の形にします．

```bash
albam serve --config albam.yml
```

## 9. アルバムごとのメタデータに対応する

各ディレクトリに `_album.yml` を置けるようにします．

例です．

```yaml
title: 京都旅行
description: 2026 年春の京都旅行
cover: cover.jpg
tags:
  - travel
  - kyoto
```

まずは，次の 3 つだけ対応すれば十分です．

```txt
title
description
cover
```

そのため，`albums` テーブルを拡張します．

```sql
ALTER TABLE albums ADD COLUMN description TEXT;
ALTER TABLE albums ADD COLUMN cover_asset_id INTEGER;
```

この段階で，ディレクトリ名そのままではなく，人間向けのタイトルを表示できるようになります．

## 10. タグを追加する

次に，画像やアルバムにタグを付けられるようにします．

最小スキーマは次です．

```sql
CREATE TABLE tags (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,
    slug TEXT NOT NULL UNIQUE
);

CREATE TABLE asset_tags (
    asset_id INTEGER NOT NULL,
    tag_id INTEGER NOT NULL,
    PRIMARY KEY (asset_id, tag_id),
    FOREIGN KEY (asset_id) REFERENCES assets(id),
    FOREIGN KEY (tag_id) REFERENCES tags(id)
);
```

追加する API はこれです．

```txt
GET /api/tags
GET /api/tags/{slug}/assets
```

タグの入力は，最初は `_album.yml` や `_assets.yml` から読むだけで十分です．Web UI から編集する機能は後回しにします．

## 11. 検索を追加する

タグやタイトルで検索できる API を追加します．

```txt
GET /api/search?q=...
```

最初は `LIKE` 検索で十分です．

対象は次のあたりです．

```txt
albums.title
albums.description
assets.filename
tags.name
```

あとから必要になったら，SQLite の FTS に移行します．

## 12. ファイル変更監視を追加する

最後に，ディレクトリの変更を監視して自動で再インデックスします．

担当パッケージは `internal/watcher` です．

```txt
internal/watcher/
└── watcher.go
```

実装には `fsnotify` を使います．

ただし，最初から監視機能を入れる必要はありません．まずは手動で十分です．

```bash
albam index
albam serve
```

後から次のようにします．

```bash
albam serve --watch
```

## 実装順まとめ

```txt
1. Go プロジェクトを作る
2. 画像ファイルの再帰的列挙を実装する
3. ディレクトリ構造から Album / Asset を作る
4. SQLite を導入する
5. albam index <dir> でスキャンと DB 保存を行う
6. albam serve で HTTP サーバーを起動する
7. GET /api/albums を作る
8. GET /api/albums/{id}/assets を作る
9. GET /api/assets/{id}/file を作る
10. サムネイル生成を追加する
11. GET /api/assets/{id}/variants/thumb を作る
12. albam.yml を導入する
13. _album.yml を読む
14. タグを追加する
15. 検索を追加する
16. fsnotify による変更監視を追加する
```

## MVP の完成条件

最初の MVP は，次ができれば十分です．

```txt
できること:
  - 指定ディレクトリ以下を再帰的に読む
  - 画像ファイルだけを認識する
  - ディレクトリ構造をアルバム階層として扱う
  - アルバム情報を SQLite に保存する
  - 画像情報を SQLite に保存する
  - API でアルバム一覧を返す
  - API で画像一覧を返す
  - API 経由で画像ファイルを返す
```

まだやらなくてよいものは次です．

```txt
後回し:
  - ログイン
  - 複数ユーザー
  - Web UI からの編集
  - 高度なタグ管理
  - Exif の完全対応
  - 顔認識
  - 地図表示
  - ファイル変更の自動監視
```

## 最初のゴール

まずは，次のコマンドを作るところから始めるのがよいです．

```bash
albam index ./albums
```

このコマンドで画像ファイルをスキャンして DB に保存できれば，その後の API 化，サムネイル生成に自然につなげられます．
